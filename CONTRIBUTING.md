### # Contributing to ReliefTrack AI

Thank you for your interest in contributing to ReliefTrack AI.

We welcome contributions that improve the platform's functionality, security, documentation, and user experience.

### ## Development Workflow

* Fork the repository.

* Create a feature branch.

* Make your changes.

* Test your changes.

* Commit using descriptive commit messages.

* Open a pull request.

### ## Branch Naming

Use descriptive branch names such as:

feature/authentication

feature/inventory-management

fix/login-validation

### ## Commit Message Convention

Examples:

feat(auth): add JWT authentication

fix(api): correct validation error handling

docs(readme): update setup instructions

### ## Code Standards

* Use Python type hints.

* Follow PEP 8 guidelines.

* Write clear and descriptive variable names.

* Add docstrings where appropriate.

* Keep functions focused on a single responsibility.

### ## Reporting Issues

When reporting a bug, include:

* A clear description of the problem

* Steps to reproduce

* Expected behavior

* Actual behavior

* Environment details (OS, Python version, Docker version)

### ## Questions and Suggestions

Constructive feedback and feature suggestions are always welcome.

Together we can build a platform that strengthens humanitarian operations through technology and responsible innovation.
